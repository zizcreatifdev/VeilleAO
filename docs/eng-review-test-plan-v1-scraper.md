## Framework
pytest + responses (HTTP mocking) + pytest-cov

## Key Modules to Test

### filter.py
- Keyword match with FR accents: "identité visuelle" matches "identite visuelle"
- Keyword match with caps: "COMMUNICATION" matches keyword "communication"
- EN keyword match on PNUD-origin AOs
- No match → returns empty list
- Partial match → returns only matching AOs
- AO with keyword only in description (not title) → included
- AO with no keywords anywhere → excluded

### dedup.py
- URL normalization: http → https
- URL normalization: trailing slash removal
- URL normalization: query param sort
- URL normalization: strip utm_source, utm_medium, fbclid, gclid
- Already-seen AO → not returned in new list
- New AO → returned + hash written to state.json
- state.json created if not exists (first run)
- state.json loaded correctly from disk
- purge_old_entries: entries >30 days removed
- purge_old_entries: entries <30 days kept
- source_failures: incremented when source flag passed
- source_failures: reset to 0 on success flag
- source_failures: alert threshold (>3) correctly detected

### sources/dcmp.py (and other HTML scrapers — same pattern)
- Mock HTTP 200 with fixture HTML → returns valid list[AO dict]
- Mock HTTP 403 → raises exception
- Mock HTTP 429 → raises exception
- Mock HTTP 503 → raises exception
- Mock timeout (ConnectionTimeout) → raises exception
- HTML structure changed (key selector not found) → returns [] or raises with useful error
- Each returned AO dict has all required fields: title, url, source, date_published, deadline, budget, description, language

### sources/pnud.py (JSON API — different from HTML scrapers)
- Mock API 200 with fixture JSON → returns valid list[AO dict]
- Mock API error → raises exception
- Unexpected JSON shape (missing key) → graceful handling, returns [] or raises

### email_digest.py
- Results template: correct subject line format, correct AO count in subject, each AO in body
- Empty template: correct subject, correct "no results" body
- Alert section: included when source_failures > threshold, excluded when below
- SMTP send: mock smtplib, verify send() called with correct arguments
- SMTP retry: first send fails → retry 1x → success
- SMTP both fail → exception logged, GH Actions exit code nonzero

### main.py
- All sources succeed → email sent, state written
- Some sources fail → email sent with available AOs, error section present
- All sources fail → degraded email sent, state written with failures
- Top-level exception (import error, config error) → try/except catches, fallback behavior

## Critical Edge Cases
- AO with missing deadline field → handled gracefully (shows "" not crash)
- AO description longer than 200 chars → truncated to 200
- state.json corrupted (invalid JSON) → handled gracefully (recreate)
- state.json missing on first run → created automatically
- RECIPIENTS list with multiple emails → all receive the digest

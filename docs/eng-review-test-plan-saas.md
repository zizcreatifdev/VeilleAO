## Affected Pages/Routes

### Sem 1 — Landing + Wave
- `/` — landing page : formulaire waitlist (nom, email, téléphone, plan)
- `/api/waitlist` POST — insert waitlist_signups + Resend confirmation email
- `/payment/confirm` — page soumission référence Wave
- `/api/payments` POST — validation regex T_[A-Z0-9]{18}, insert payments, email admin
- `/api/cron/check-pending-payments` POST — relance 48h (Vercel Cron, protégé CRON_SECRET)

### Sem 2-3 — Admin Dashboard
- `/admin` — liste agences, paiements en attente, stats
- `/admin/payments/[id]` — bouton "Confirmer" paiement Wave
- `/api/admin/payments/[id]` PATCH — confirmation: update statut + agency.status + Clerk Org (Sem 3) + email client

### Sem 3-5 — Agency Dashboard
- `/agency/dashboard` — liste AOs filtrés par keywords
- `/api/agency/aos/[id]` PATCH — marquer AO (interesting/ignored/applied)
- `/api/agency/preferences` PATCH — update extra_keywords

## Key Interactions to Verify

### Critiques (E2E Playwright)
- Flow Wave complet : form → Wave link → submit référence → page confirmation
- Admin confirmation : login admin → voir paiement en attente → cliquer Confirmer → agence reçoit email
- Cron relance : simuler paiement > 48h → appeler cron endpoint → email relance envoyé (mock Resend)

### Unit tests (Vitest)
- `/api/payments` : regex valide, regex invalide (12 chars, lowercase, sans T_), UNIQUE violation → 409
- `/api/admin/payments/[id]` PATCH : auth check (non-admin → 403), double-confirm → idempotent
- `/api/cron/check-pending-payments` : CRON_SECRET absent → 401, valide → 200
- Middleware Clerk : /admin sans role:owner → redirect, /agency sans org → redirect /sign-in
- extra_keywords validation : format valide, format invalide (HTML injection, SQL-like)

## Edge Cases

- Double-clic sur "Soumettre référence Wave" → UNIQUE constraint renvoie 409, pas deux payments créés
- Formulaire waitlist avec email déjà existant → message clair (409 ou 200 silencieux ?)
- Admin confirme un paiement déjà confirmé → idempotent (ne recrée pas la Clerk Org, ne renvoie pas l'email)
- Supabase down lors du write scraper → best-effort, digest email envoyé quand même
- Cron appelé sans CRON_SECRET header → 401 immédiat, pas d'email envoyé
- extra_keywords avec 100 mots → performance ILIKE acceptable (< 2s)
- appels_offres avec 0 résultats pour les keywords → dashboard vide, message "Aucun AO aujourd'hui"

## Critical Paths (E2E obligatoire)

1. **Wave payment complet** : form waitlist → email confirmation → lien Wave → saisie T_XXXX → email admin → admin confirme → email agence → agence accède au dashboard
2. **Double-submit prevention** : soumettre la même référence Wave deux fois → deuxième tentative → 409, pas de doublon en DB
3. **Admin auth** : utilisateur sans publicMetadata.role=owner accède /admin → redirect /
4. **Agency isolation** : agence A ne voit pas les org_seen_aos de l'agence B (RLS Supabase)
